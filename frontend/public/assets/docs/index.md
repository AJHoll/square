## Общая информация

**Скверъ** предназначен для облегчения работы экпертов и участников во время проведения мероприятий и
чемпионатов AtomSkills и подобных.
Ниже представлена документация по основному процессу работы с системой.

## Добавление пользователей

Первое, что нужно сделать после развертывания системы - добавить пользователей. Убедитесь, что вы зашли
в систему под пользователем Администратор. Стандартный пароль от администратора **ghjcnjq**.

**После развертывания,мы настоятельно рекомендуем сменить пароль от пользователя администратора в профиле.**

![Добавление пользователя - пункт меню](./img/000_пункт_меню_профиля.png)

Для добавления пользователей перейдем в пункт меню **Управление пользователями**

![Добавление пользователя - пункт меню](./img/001_добавление_пользователя_1.png)

Итак, добавить пользователей можно двумя способами:

1. С помощью формы заведения нового пользователя (кнопка **Создать**)
2. Импорт пользователь из файла .xlsx

### Создание пользователя через форму

Для первого способа нужно нажать на кнопку **Создать**

По нажатию кнопки возникнет диалог
![Диалог создания пользователя](./img/002_диалог_создания_пользователя.png)

Вводим необходимые данные:

* **Системное имя** - оно используется как логин при входе. Должно содержать только английские буквы
* **Наименование** - это ФИО пользователя.
* **Пароль** - это базовый пароль пользователя для первого входа. Мы настоятельно рекомендуем оповещать пользователя о
  необходимости его смены после входа в систему.

После заведения данных нужно нажать на кнопку **Сохранить**.
После чего форма будет закрыта и новый пользователь появится в таблице пользователей.

### Создание пользователей через импорт данных

Для создания больших количеств пользователей сразу мы рекомендуем пользоваться импортом через xlsx
Для этого вам следует скачать шаблон для импорта
![Скачать шаблон импорта пользователей](./img/003_скачать_шаблон_импорта_пользователей.png)

После чего на ваш компьютер будет скачан файл **Скверъ__шаблон_импорта_пользователей.xlsx**
![Структура файла импорта пользователей](./img/004_структура_файла_импорта_пользователей.png)

Здесь так же нужно заполнить требуемые поля:

* **Логин** - оно используется как логин при входе. Его можно опустить, **но ФИО в таком случае надо указать полностью**
* **ФИО** - ФИО пользователя. **Обязательно к заполнению**
* **Пароль по-умолчанию** - это базовый пароль пользователя для первого входа. **Обязательно к заполнению**

После заполнения шаблона его следует импортировать в систему
![Импорт файла с пользователями](./img/005_импорт_файла_с_пользователями.png)

После этого действия данные о пользователях будут внесены в систему.

* **Поскольку импорт не грузит дубли, можно пользоваться одним файлом и дополнять его по мере необходимости.**

## Управление площадками

### Добавление площадки

Для добавления площадок следует выбрать пункт меню **Управление площадками**
![Управление площадками](./img/006_управление_площадками.png)

В открывшемся окне нажать на кнопку **Создать**

По нажатию на кнопку откроется форма добавления новой площадки
![Форма добавления новой площадки](./img/007_окно_содания_площадки.png)

В нем требуется заполнить 3 поля:

* **Системное имя** - Название площадки на английском языке (обычно название компетенции на англ. языке) **Обязательное
  **
* **Наименование** - Название площадки (оно же название компетенции) **Обязательное**
* **Описание** - Более полное описание площадки/компетенции (не обязательно к заполнению)

После заполнения формы при нажатии на кнопку **Сохранить** площадка будет создана.

Для назначения ролей нужно пройти на вкладку **Роли на площадке** в окне **Управление площадками**
![008_роли_на_площадке_таб](./img/008_роли_на_площадке_таб.png)

Для изменения состава роли следует нажать на кнопку **Редактировать**
![009_изменить_роли_на_площадке](./img/009_изменить_роли_на_площадке.png)

Выберите нужного пользователя и нажмите на кнопку добавления (зеленый плюс)
![010_добавление_пользователя_в_роль_площадки](./img/010_добавление_пользователя_в_роль_площадки.png)

После чего у данного пользователя сменится состояние в колонке **Используется**

После назначения всех требуемых пользователей на роли, следует выйти из режима редактирования для просмотра

![013_возвращение_к_просмотру_состава_ролей_площадки](./img/013_возвращение_к_просмотру_состава_ролей_площадки.png)

### Управление критериями

Далее следует определиться с критериями. Для этого следует зайти в окно **Управление критериями**
![034_управление_критериями](./img/034_управление_критериями.png)

После входа в окно управления критериями есть два способа для заведения критериев в систему:

1. Ручной ввод критериев
2. Импорт критериев из платформы через стандартный excel файл

#### Ручной ввод критериев и изменение

Для ручного ввода критерия нажмите соответствующую кнопку в интерфейсе
![035_ручной_ввод_критерия_критерий](./img/035_ручной_ввод_критерия_критерий.png)

После добавления заполните необходимые поля, после чего заведите субкритерии с помощью одноименной кнопки
![036_ручной_ввод_критериев_субкритерий](./img/036_ручной_ввод_критериев_субкритерий.png)

Для добавления аспектов так же существует кнопка в форме субкритериев
![037_ручной_ввод_критериев_аспект](./img/037_ручной_ввод_критериев_аспект.png)

Для удаления любого элемента (сверху-вниз: критерий, субкритерий, аспект) есть кнопка с изображением урны красного цвета
![039_удаление_элементов_критериев](./img/039_удаление_элементов_критериев.png)

Система работает с 4 типами критериев

**Бинарный**
![038_ручной_ввод_критериев_тип_аспекта](./img/038_ручной_ввод_критериев_тип_аспекта.png)

**Дискретный**
![040_дискретный_аспект](./img/040_дискретный_аспект.png)

**Судейский**
![041_судейский_аспект](./img/041_судейский_аспект.png)

**Отсекаюший**
![042_отсекающий_оспект](./img/042_отсекающий_оспект.png)

Любой аспект в управлении критериев можно связать с любым отсекающим
аспектом. При этом реализован выбор отсекающих аспектов в формате (субкритерий - имя отсекающего аспекта) для
удобства навигации. Для отключения связи с отсекающим критерием, следует нажать кнопка разлинковать, в правой части
формы аспекта.

Для каждого критерия считается сумма его аспектов. Если сумма аспектов не равна весу критерия, то поле ввода
подсветится красным, так как это ошибка и такого быть не должно.
![043_сумма_аспектов_по_критериям](./img/043_сумма_аспектов_по_критериям.png)

Система так же поддерживает проверку критериев по модулям, для чего в критериях, субкритериях и аспектах
присутствуют соответствующие поля.
![043_сумма_аспектов_по_критериям](./img/047_модули_в_критериях.png)

Мы рекомендуем проставлять модули после всех прочих изменений. Важно, что изменение поля модуль, распространяется
на все его дочерние записи, поэтому будьте предельно внимательны при их простановке.

Для использования, готовые критерии должны быть сохранены. Для чего предусмотрена одноименная кнопка
![044_сохранение_аспектов](./img/044_сохранение_аспектов.png)

Мы так же реализовали возможность выгрузить данные в шаблоне платформы, для более удобного формирования критериев.

### Группы проверки и проверяемые модули

После заведения критериев и их сохранения (данные в листы проставляются из сохраненных данных системы) возможно
сформировать
листы проверки на каждую из заведенных команд.
![049_формирование_листов_проверки](./img/049_формирование_листов_проверки.png)

Для того, чтобы начать процедуру проверки следует вернуться в окно управление площадками и завести группы проверки .
![049_формирование_листов_проверки](./img/050_управление_площадками_вкл_группы_проверки.png)

Для создания группы проверки нажмите кнопку **Создать** во вкладке **Группы проверки**. Откроектся форма заполнения
группы проверки
![051_форма_создания_группы_проверки](./img/051_форма_создания_группы_проверки.png)

В форме нужно заполнить следующие поля:

* **Код** - системное имя группы проверки (латиницей) **обязательное**
* **Наименование** - наименование группы проверки на русском языке **обязательное**
* **Модули** - это проверяемые группой проверки модули. Перечисляются через запятую без пробелов. (например: 1,2,3)

После создания, следует заполнить группу проверки экспертами. Для этого нажмите кнопку **Редактировать** в правой части
экрана
![052_формирование_состава_группы_проверки](./img/052_формирование_состава_группы_проверки.png)

После чего выберите нужного эксперта и нажмите на кнопку **Добавить** (зеленый плюс)
![053_добавление_эксперта_в_группу_проверки](./img/053_добавление_эксперта_в_группу_проверки.png)

Если нужно убрать эксперта из группы проверки - нажмите на **Убрать** (красный минус)
![054_удаление_эксперта_из_группы_проверки](./img/054_удаление_эксперта_из_группы_проверки.png)

Так же в самой компетенции (площадке) нужно установить, какие модули в данный момент можно проверять
(если задание выполняется частями, то каждый день можно добавлять по 1 модулю).
Делается это в поле **Проверяемы модули**в форме редактирования площадки.
![055_добавление_текущих_проверяемых_модулей](./img/055_добавление_текущих_проверяемых_модулей.png)

### Заведение оценок

После всех вышеперечисленных действий у экспертов группы проверки откроется возможность заводить оценки. 
Для этого им нужно перейтив пункт меню **Заведение оценок**
![056_заведение_оценок_пм](./img/056_заведение_оценок_пм.png)

Далее откроется форма, в которой нужно выбрать группу проверки (если вдруг эксперт входит в несколько). Проверяемый
модуль и проверяемую команду. После чего отобразится список аспектов доступных для проверки.
![057_окно_заведения_оценок](./img/057_окно_заведения_оценок.png)

Важно отметить, что в системе реализованы отсекающие аспекты и они так же работают в заведении.
При вводе положительного исхода в отсекающем аспекте, все связанные с ним аспекты будут обнулены, а ввод
оценок для них будет заблокирован. Так же система подскажет, в связи с каким критерием была проведена
блокировка.
![058_работа_с_отсекающими_аспектами](./img/058_работа_с_отсекающими_аспектами.png)

После введения оценок (мы рекомендуем производить это действие сильно чаще) требуется сохранить полученные оценки.
Для этого предусмотрена кнопка **Сохранить**.
![059_сохранение_оценок](./img/059_сохранение_оценок.png)

После сохранения ГЭ компетенции может экспортировать данные в эксель для заведения их в платформу (этим же эксель файлом)
![061_экспорт_в_эксель_оценок_кнопка](./img/061_экспорт_в_эксель_оценок_кнопка.png)

По нажатию на кнопку скачается файл следующего формата
![060_форма_выгрузки_оценок_в_эксель](./img/060_форма_выгрузки_оценок_в_эксель.png)
